# 정부 지원사업 정보 수집 & AI 요약 봇 — 리서치 문서

> 작성일: 2026-03-01
> 프로젝트명: GovBot (가칭)

---

## 1. 프로젝트 개요

정부 각 부처, 지자체, 공공기관에 흩어져 있는 지원사업 공고 정보를 자동으로 수집하고, AI로 핵심 내용을 요약하여 사용자에게 알림으로 전달하는 스케줄러 기반 봇.

### 1.1 해결하려는 문제

- 지원사업 공고가 기업마당, 보조금24, K-Startup, 각 지자체 등 **여러 플랫폼에 분산**되어 있어 일일이 확인하기 어려움
- 공고문이 길고 복잡하여 **핵심 정보(대상, 금액, 마감일)를 파악하는 데 시간이 많이 소요**
- 마감일을 놓치거나, 자신에게 맞는 지원사업이 있는지조차 모르고 지나치는 경우가 빈번
- 기존 서비스들은 단순 목록 나열에 그치며, **개인 맞춤형 필터링 + AI 요약**이 결합된 서비스가 부족

### 1.2 핵심 기능 우선순위 (사용자 설정 기준)

1. **신규 공고 실시간 알림** — 새로운 공고 감지 시 즉시 알림
2. **내 조건에 맞는 자동 매칭** — 키워드, 분야, 지역 기반 필터링
3. **공고 내용 AI 요약** — LLM 기반 핵심 정보 추출
4. **마감일 관리 & 리마인더** — D-day 카운트다운, 사전 알림

## 2. 데이터 소스 분석

### 2.1 기업마당 (bizinfo.go.kr) — ⭐ 핵심 1순위

**중소벤처기업부**가 운영하는 중소기업 정책 포털로, 정부부처/지자체/유관기관의 지원사업 공고를 통합 수집하여 제공한다.

**API 정보:**

| 항목 | 내용 |
|------|------|
| URL | `https://www.bizinfo.go.kr/uss/rss/bizinfoApi.do` |
| 방식 | GET |
| 응답 형식 | JSON, XML (선택 가능) |
| 인증 | API 인증키 필요 (`crtfcKey` 파라미터) |
| 발급 방법 | 기업마당 사이트에서 사용 신청 → 이메일로 키 발급 |

**API 파라미터:**

| 파라미터 | 설명 |
|---------|------|
| `crtfcKey` | API 인증키 (필수) |
| `keyword` | 검색 키워드 (선택) |
| `dataType` | 응답 형식 - XML 또는 JSON |
| 해시태그 필터 | 금융, 기술, 인력, 수출, 내수, 창업, 경영, 기타 |
| 지역 필터 | 서울~제주 17개 시도 |

**제공 데이터 필드:**

- 분야 (8종: 금융, 기술, 인력, 수출, 내수, 창업, 경영, 기타)
- 사업명
- 신청시작일자 / 신청종료일자
- 소관기관 (중앙행정기관, 지자체 등)
- 수행기관 / 접수기관
- 등록일자
- 상세URL (공고 원문 링크)

**장점:** 중앙부처 + 지자체 + 유관기관 공고를 이미 통합 수집하고 있으므로, 이 API 하나로 대부분의 지원사업 공고를 커버할 수 있다. GPTs 연동 사례도 존재하여 검증된 API.

**단점:** 인증키 발급 시 IP 또는 시스템 URL 등록 필요. 일부 상세 공고 내용(본문 전체)은 API에 포함되지 않아, 상세URL을 통한 추가 크롤링이 필요할 수 있음.

### 2.2 보조금24 (gov.kr/보조금24) — ⭐ 핵심 2순위

**행정안전부**가 운영하는 대한민국 공공서비스 정보 플랫폼. 정부 각 부처, 지자체, 공공기관, 교육청의 수혜 서비스 정보를 제공한다.

**API 정보:**

| 항목 | 내용 |
|------|------|
| 데이터명 | 행정안전부_대한민국 공공서비스(혜택) 정보 |
| 제공처 | 공공데이터포털 (data.go.kr) |
| 신청 | data.go.kr에서 활용 신청 → 인증키 발급 |

**특징:**

- 1,075개 이상의 중앙부처 보조금 정보를 오픈API로 공개
- 기업 지원뿐 아니라 **개인 대상 복지/보조금** 정보도 포함
- 이미 민간 앱에서 활용 중 (App Store에 '보조금24' 관련 앱 존재)
- 기업마당이 주로 기업 지원사업에 특화된 반면, 보조금24는 더 넓은 범위의 공공 혜택을 커버

### 2.3 공공데이터포털 (data.go.kr) — 보조 소스

정부 및 공공기관의 모든 데이터를 통합 제공하는 중앙 플랫폼으로, 10만 개 이상의 데이터셋을 보유하고 있다.

- 기업마당, 보조금24의 API 인증키 발급 창구 역할
- 자동 변환 기능: 3단계 이상 오픈 포맷 파일데이터를 RestAPI 기반 JSON/XML로 자동 변환 제공
- 활용 신청 후 **1~2시간 대기** 필요 (한국 공공데이터 포털의 특이한 점)

### 2.4 K-Startup (k-startup.go.kr) — 보조 소스

**중소벤처기업부** 산하 창업 지원 포털. 창업 단계별/관심 분야별 사업 공고를 분류하여 제공한다. 별도 API 제공 여부는 확인 필요하며, 기업마당과 데이터가 상당 부분 겹침.

### 2.5 중소벤처24 (smes.go.kr) — 보조 소스

중기부 유관기관의 공고정보를 통합적으로 제공하는 API를 별도 보유. 증명(확인)서 정보 API도 제공하여 자격 검증 연계도 가능하다.

### 2.6 데이터 소스 전략 요약

```
[우선순위]
1차: 기업마당 API  →  전체 분야 지원사업 공고 통합 수집
2차: 보조금24 API  →  개인 대상 보조금/복지 혜택 보완
3차: 상세URL 크롤링 →  공고 본문 텍스트 추출 (AI 요약용)
```

> **핵심 판단:** 기업마당 API만으로도 MVP(최소 기능 제품)는 충분히 만들 수 있다. 보조금24는 커버리지 확대 시 2단계로 추가.

## 3. 기술 아키텍처 설계

### 3.1 전체 흐름

```
[스케줄러] → [데이터 수집] → [중복 체크] → [AI 요약] → [매칭 필터] → [알림 발송]
   │              │               │              │              │              │
 Cron/          API 호출        DB 비교       LLM API       키워드/       Discord/
 GitHub       + 크롤링        (신규만)       (Claude/      분야 필터    Telegram
 Actions                       통과          GPT 등)                   Webhook
```

### 3.2 기술 스택 옵션

**A안: GitHub Actions 기반 (서버리스, 무료) — 추천**

| 구성요소 | 기술 |
|---------|------|
| 스케줄러 | GitHub Actions cron |
| 언어 | Python 3.x |
| 데이터 수집 | requests + BeautifulSoup4 |
| 데이터 저장 | JSON 파일 (GitHub repo 내) 또는 SQLite |
| AI 요약 | Anthropic Claude API / OpenAI API |
| 알림 | Discord Webhook / Telegram Bot API |
| 배포 | GitHub repository (자동 실행) |

장점: 서버 비용 0원, 별도 인프라 불필요, Git으로 버전 관리, 코드 push만으로 배포 완료.

단점: GitHub Actions 무료 한도 (월 2,000분, public repo는 무제한), cron 실행 시간에 약간의 지연 (5~15분) 발생 가능, 상시 서버가 아니므로 실시간성은 제한적.

**B안: 로컬 PC + APScheduler (간단, 개인용)**

| 구성요소 | 기술 |
|---------|------|
| 스케줄러 | APScheduler (Python 라이브러리) |
| 언어 | Python 3.x |
| 데이터 저장 | SQLite / JSON 파일 |
| AI 요약 | Claude API |
| 알림 | Discord Webhook |

장점: 설정 간단, 로컬에서 즉시 테스트/실행, 완전한 제어.

단점: PC를 계속 켜놔야 함, 외출 시 작동 불가.

**C안: 클라우드 VPS (장기 운영용)**

Oracle Cloud / AWS Free Tier + Ubuntu + crontab 조합. 24시간 상시 운영이 가능하지만 초기 세팅이 비교적 복잡.

### 3.3 권장안

**1단계:** A안(GitHub Actions)으로 시작 → MVP 빠르게 구축, 무료 운영
**2단계:** 필요 시 C안으로 마이그레이션 → 실시간 모니터링, DB 고도화

## 4. 핵심 모듈별 상세 설계

### 4.1 데이터 수집 모듈

```python
# 기업마당 API 호출 예시 구조
import requests

def fetch_bizinfo(api_key, keyword=None, data_type="JSON"):
    url = "https://www.bizinfo.go.kr/uss/rss/bizinfoApi.do"
    params = {
        "crtfcKey": api_key,
        "dataType": data_type,
    }
    if keyword:
        params["keyword"] = keyword
    
    response = requests.get(url, params=params)
    return response.json()  # 또는 XML 파싱
```

상세 공고 본문이 필요한 경우, 반환된 `상세URL`을 추가 크롤링:

```python
from bs4 import BeautifulSoup

def fetch_detail(detail_url):
    response = requests.get(detail_url)
    soup = BeautifulSoup(response.text, 'html.parser')
    # 공고 본문 텍스트 추출
    content = soup.select_one('.공고본문_셀렉터')
    return content.get_text(strip=True)
```

### 4.2 중복 체크 모듈

이전에 수집한 공고와 비교하여 신규 공고만 처리하는 로직. 공고 고유 ID 또는 제목+등록일 조합으로 판별.

```python
import json

def load_seen_ids(filepath="seen_ids.json"):
    try:
        with open(filepath, 'r') as f:
            return set(json.load(f))
    except FileNotFoundError:
        return set()

def save_seen_ids(seen_ids, filepath="seen_ids.json"):
    with open(filepath, 'w') as f:
        json.dump(list(seen_ids), f)

def filter_new(announcements, seen_ids):
    new_ones = []
    for item in announcements:
        uid = item.get('pblancId') or item.get('title')
        if uid not in seen_ids:
            new_ones.append(item)
            seen_ids.add(uid)
    return new_ones
```

### 4.3 AI 요약 모듈

Claude API를 활용하여 공고 본문에서 핵심 정보를 구조화하여 추출.

```python
import anthropic

def summarize_announcement(title, content):
    client = anthropic.Anthropic(api_key="YOUR_KEY")
    
    prompt = f"""다음 정부 지원사업 공고를 분석하여 아래 형식으로 요약해주세요:

📌 사업명: 
🏢 소관기관: 
🎯 지원대상: 
💰 지원내용/금액: 
📅 신청기간: 
📋 신청방법: 
⚠️ 유의사항: 
🔗 한줄요약: 

---
공고 제목: {title}
공고 내용: {content}
"""
    
    response = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=1000,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text
```

### 4.5 알림 발송 모듈

```python
import requests

def send_discord(webhook_url, message):
```

```python
import telegram
```

### 4.6 GitHub Actions 워크플로우

```yaml
name: gov_support_bot
on:
  schedule:
    - cron: '0 0,6 * * *'   # 매일 오전 9시, 오후 3시 (KST = UTC+9)
  workflow_dispatch:         # 수동 실행도 가능

  run-bot:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python

        with:
          python-version: '3.11'
      - name: Install dependencies

      - name: Run bot

          BIZINFO_API_KEY: ${{ secrets.BIZINFO_API_KEY }}

          DISCORD_WEBHOOK: ${{ secrets.DISCORD_WEBHOOK }}
        run: python main.py
      - name: Commit updated data
        run: |
          git config --global user.name 'Bot'
          git config --global user.email 'bot@github.com'
          git add seen_ids.json
          git commit -m "Update seen IDs" || true
          git push
```

## 6. 유사 프로젝트 & 선행 사례

### 6.1 GPTs + 기업마당 API 연동

GPTers 커뮤니티에서 기업마당 API를 활용한 GPTs 제작 사례가 공유되어 있다. OpenAPI 스키마를 GPTs의 Action 기능에 등록하여 대화형으로 지원사업을 검색하는 방식. 우리 프로젝트와 다른 점은, GPTs는 사용자가 직접 질의해야 하지만 우리 봇은 **자동으로 수집+알림**한다는 것.

### 6.2 나라장터 크롤러 (GitHub: seoweon/narajangteo)

입찰공고를 키워드로 필터링하여 엑셀로 정리하는 Python 스크립트. BeautifulSoup 없이 API 기반으로 동작. 키워드 리스트 + 제외 키워드 설정 구조가 참고할 만함.

### 6.3 텔레그램 크롤링 알림봇 패턴

웹 크롤링 → 이전 데이터와 비교 → 변경 감지 시 텔레그램 메시지 발송하는 패턴은 이미 많은 한국어 튜토리얼에서 검증된 구조. crontab 또는 GitHub Actions로 스케줄링.

### 6.4 보조금24 활용 앱 (App Store)

행안부의 보조금24 API를 직접 연동하여 개인별 맞춤형 보조금 검색 기능을 제공하는 민간 앱이 이미 App Store에 존재. API 활용 가능성과 데이터 품질이 검증된 셈.

## 7. 구현 로드맵

### Phase 1: MVP (1~2주)

- 기업마당 API 인증키 발급 및 연동
- Python 스크립트: API 호출 → JSON 파싱 → 신규 공고 필터링
- Discord Webhook으로 알림 발송
- GitHub Actions로 하루 2회 자동 실행
- 아직 AI 요약 없이, 제목 + 마감일 + 링크만 전달

### Phase 2: AI 요약 추가 (2~3주)

- 상세URL 크롤링으로 공고 본문 수집

## 8. 리스크 & 고려사항

### 8.1 API 안정성

공공데이터 API는 간헐적으로 응답이 느리거나 서버 점검으로 중단될 수 있다. 재시도 로직(retry)과 에러 핸들링이 필수. 특히 공공데이터포털은 활용 신청 후 인증키 활성화에 1~2시간이 걸리는 점 유의.

### 8.2 크롤링 윤리 & 법적 이슈

기업마당, 보조금24 모두 공식 API를 제공하므로 API를 사용하는 한 법적 문제 없음. 단, 상세 공고 페이지 크롤링 시에는 robots.txt 확인, 적절한 요청 간격 유지(1~2초), User-Agent 명시 등 기본적인 크롤링 에티켓을 준수해야 함.

### 8.3 AI 요약 품질

공고문은 행정 용어가 많고 구조가 제각각이므로, 프롬프트 엔지니어링이 중요. 요약 품질을 일정하게 유지하려면 few-shot 예시를 프롬프트에 포함하는 것이 효과적.

### 8.4 비용 관리

AI 요약은 건당 비용이 발생하므로, 모든 공고를 요약하기보다 **매칭 필터를 먼저 적용** → 관심 공고만 AI 요약하는 순서로 처리하면 비용을 효과적으로 관리할 수 있다.

### 8.5 데이터 중복

기업마당과 보조금24의 데이터가 일부 겹칠 수 있으므로, 사업명+소관기관 조합으로 중복 제거 로직이 필요.

## 10. 결론

이 프로젝트는 **기업마당 공식 API**를 핵심 데이터 소스로, **GitHub Actions**를 스케줄러로, **Claude API**를 요약 엔진으로 조합하면 최소 비용(월 $3~10)으로 실용적인 봇을 구축할 수 있다.

기존 아티스트 인텔리전스 봇 개발 경험에서 사용한 "공개 데이터 자동 수집 → 구조화 → 알림" 패턴과 거의 동일한 구조이므로, 기술적 허들은 높지 않다. 핵심은 API 인증키 발급과 프롬프트 엔지니어링(요약 품질 확보)에 있다.

MVP는 1~2주 내에 동작하는 수준까지 빠르게 만들 수 있으며, 이후 AI 요약과 매칭 기능을 점진적으로 고도화하는 전략이 가장 현실적이다.
